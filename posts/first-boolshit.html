第一篇随便写写，就从这开始吧。

2022-06-01
生活

现在是2022年的6月1号凌晨1点30分，是的，我又在熬夜了。

上一次不熬夜，我不记得是什么时候了。其实也想不通为什么熬夜，该做的工作也已经做完了，没做完的工作，也不一定要熬夜把它做完。

这段时间，特别是五月份，不知为何总是感觉内心疲倦。说不上是哪里的原因，压力太大可能也算不上，有许多的人比我压力大得多。要说是为未来担忧，也算不上，未来这种事谁说得准，变数太多了。

以前我会担心各种，担心以后担心未来。但是慢慢随着年龄的增长，这种担忧越来越淡薄。可能是见得多，也可能是想通了。人活着一辈子，到底是为了什么在活着。现在我觉得人生是毫无意义的，来到这个世界上，人就像是一粒微小的尘埃，什么都算不上。

跑题了，说说为什么做这个博客吧。

其实做博客网站，这个想法在我还在高中的时候就已经有了，至于为什么一直没能坚持到现在，还不是因为 三分钟热度 呗。

早在我初中上学的时候，那时候还没有真正的智能手机，绝大部分人还是使用的各种：直板诺基亚、翻盖手机那些。我把这些叫做半智能手机，因为他们也支持上网功能。那时候上网流量是按照几十M，几百M计算的，网页往往也只有文字，看一个完整的网页，你需要按下翻键很多次。

大家都在研究用手机看小说，玩QQ家园的时候，我就申请了好几个域名，不过都是免费的。如果你有过了解的话，是的，就是TK域名。但是建设网站，还需要一个网站空间，学生嘛没有钱，所以就只能拿着申请的域名自嗨了。逛过很多论坛，当过各种版主，我暗自发誓以后要做一名 站长 。后来“站长梦”也实现了，但是是寄人篱下，自助建站的。何谓自助建站：就是你在别人网站进行申请，通过后会分配给你一个二级域名，当作你自己的网站。

真正到了我拥有自己域名的时候，可能已经到了高中辍学后。有了一点“经济实力”，买了自己的第一个CC域名，当时CC域名还挺火的，不会像现在因为它价格便宜，被很多人用来做不法网站。我用自己的姓名拼音注册的，如果那时候注册的是.com的话，根据短双拼的行情，几年后都还能大赚了，当时就光觉得CC后缀好看。

再后来注册了十来个左右的域名，要么想着用来投资，要么想着用来做一番大事业。但是事与愿违，注册的域名都是用来交学费的，自己觉得含义很好，但是根本毫无作用。至于用来做大事业的想法，都被各种搁置了，现在想来真的是没有执行力，如果想到就去做的话，已经吃过好几波的红利了，这个结论到后来都是经过验证的。

回到现在，说一下做这个博客的原因吧。

  • 记录生活。

    也许是觉得时间过得太快了，脑容量就那么大，年龄的增长，就会溢出了。所以需要记录下来，留着以后看一看，一定会回味无穷。

  • 记录成长。

    很多工作中，使用到的技巧和方法，把它们记录下来，方便自己的同时也可能会帮助到别人。

  • 完成未完成的想法

    有时候会有各种想法，要么拖延要么忘记，久而久之拖延症就越来越严重，希望以此能让自己有所改变。

所以在今年4-5月份，我在自己现有的技术基础上，额外学习了一些新的知识：

  1. Markdown
  2. TypeScript
  3. VuePress2
  4. Git工具的使用

历时20天左右，边学边用，谷歌百度过很多，感谢那些乐于分享的人。在 VuePress 这个静态网站生成工具的基础上，开发了博客的主题，截至到现在我还没有做页面主题色和一些小的效果，等到适宜的时机，我可能会把它分享出来（管它有没有人喜欢呢）。

为什么使用静态博客，而不是使用类似 Wordpress，Typecho 这样的基于Php的动态博客程序呢。相反他们是很优秀的Php网站程序，在全球有最少30%的网站都是使用的 Wordpress，同样它也是我使用的第一个网站程序。

更不是因为他们满足不了我的需求，而是因为，维护服务器是一项特别耗费精力的事情。你需要保证数据的备份，保证服务器没有漏洞，或者别忘记续费~。

为什么没有选择专为博客而生，生态更好的 Hexo 呢？

  1. 我一直想自己开发主题，这样的话可以根据需要进行设计和定制。
  2. 用 Hexo 的话，因为完全没了解过，又要增加学习和时间成本了。
  3. 使用 VuePress 可以同步学习 Vue3 和 TypeScript，简直一举两得。

至于生成速度，目前还不知道孰优孰劣，因为内容现在还没写几篇呢。平时写文章就跟你打字是一样的，不用去在意字体应该多少px，什么颜色，使用 Markdown 还能方便迁移。如果有自定义的数据，比如分类和标签，你需要放在 FrontMatter 里就可以，换网站程序的话还能方便适配。

一行命令就能把你的网页上传到如： GitHub Gitee 这类网站上，而且还不需要域名，不需要服务器，不用去担心数据和攻击风险，让你专心记流水账。

唯一的不足在我看来是速度，如果你对这个没有要求的话，那么静态博客你值得尝试，但是前提你需要有一定的电脑基础。
